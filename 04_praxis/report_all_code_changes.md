# 포팅/수정 시 변경 사항을 반드시 보고하라

코드를 포팅하거나 수정할 때, 원본 대비 변경한 모든 항목을 사용자에게 명시적으로 보고해야 한다.
"포팅 완료"라고 전달하면 상대방은 원본 그대로 온 줄 안다.

## 왜?
- vt_config_test 포팅 시 AudioSource, threshold, EMA, 추론 간격 등을 보고 없이 변경
- 사용자가 "포팅 완료"로 팀원에게 전달 → 팀원은 원본 그대로인 줄 알고 성능 측정
- 결과가 원본과 달라서 문제 발생

## 규칙
1. **작업 완료 시** "원본 대비 변경 사항" 목록을 반드시 제시
2. 변수 값, 설정, 도구, AudioSource 등 **원본과 다른 부분은 전부** 명시
3. 불가피하게 바꿨으면 바꾼 이유와 함께 **즉시 보고**
4. "동일하게 포팅했습니다"는 **진짜 동일할 때만** 사용
