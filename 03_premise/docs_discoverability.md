# 문서는 존재하는 것만으로는 부족하다 — 발견 가능해야 한다

## 원칙

하위폴더에 문서가 있는 건 구조상 맞지만, **유저가 레포에 들어왔을 때 그 문서의 존재를 인지할 수 있어야 한다.**

## 규칙

1. **모든 폴더에 README.md**: GitHub은 README.md만 자동 렌더링. 폴더마다 README.md가 있어야 유저가 그 폴더에 뭐가 있는지 안다.

2. **상위 README에서 하위 문서를 링크**: 하위폴더에 상세 문서가 있으면, 상위 README.md에 링크와 한 줄 설명을 넣어야 한다. 유저가 깊이 들어가지 않아도 어디에 뭐가 있는지 파악 가능.

3. **SDK/공통 문서는 상위에 배치**: 특정 모델에 종속되지 않는 문서(양자화 도구 비교, 공통 설정 등)는 `docs/` 같은 상위 디렉토리에 둔다. 모델 하위폴더에 넣으면 그 모델을 보는 사람만 발견함.

4. **README.md가 아닌 .md 파일은 반드시 링크**: `RESULTS.md`, `CHANGELOG.md` 등 README가 아닌 문서는 GitHub에서 자동 표시 안 됨. 반드시 README.md에서 링크해야 함.

## 체크리스트 (문서 작성 시)

- [ ] 이 문서가 들어갈 폴더에 README.md가 있는가?
- [ ] 상위 폴더 README.md에서 이 문서로의 링크가 있는가?
- [ ] 이 문서가 특정 모델/프로젝트에 종속적인가, 아니면 공통인가?
